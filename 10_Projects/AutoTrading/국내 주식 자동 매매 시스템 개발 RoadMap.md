## Stage 0. 프로젝트 기반 구축

본격적인 4단계에 들어가기 전, 반드시 먼저 해야 하는 기반 단계입니다.

|Phase|목표|담당 도구|산출물|
|---|---|---|---|
|Phase 0-1|프로젝트 원칙 정의|Obsidian|프로젝트 헌장|
|Phase 0-2|GitHub Repo 생성|Codex CLI|기본 폴더 구조|
|Phase 0-3|AGENTS.md 작성|Codex CLI|Codex 작업 규칙|
|Phase 0-4|CLAUDE.md 작성|Claude Code|AI Berkshire 사용 규칙|
|Phase 0-5|설정 파일 정의|Codex CLI|`config/*.yaml`|
|Phase 0-6|데이터 저장 정책 정의|Obsidian/Codex|raw/interim/processed 규칙|

이 단계에서 가장 중요한 산출물은 `AGENTS.md`와 `CLAUDE.md`입니다.

`AGENTS.md`에는 Codex가 지켜야 할 개발 규칙을 둡니다.

```
- 금융 데이터는 임의 생성 금지- 모든 계산은 테스트 코드 포함- 룩어헤드 바이어스 방지- raw data는 수정 금지- 모든 스크리너 기준은 config에서 관리- 투자 판단 문구는 리서치 결과로만 표현
```

`CLAUDE.md`에는 Claude Code와 AI Berkshire 리서치 규칙을 둡니다.

```
- AI Berkshire는 리서치 전용으로 사용- 매수/매도 지시 금지- 모든 핵심 수치는 출처 표시- 불확실한 데이터는 확인 불가로 표기- 국내 주식 분석 시 DART/KRX/기업 IR 우선- 최종 결과는 Markdown 리포트로 저장
```

# Stage 2 — AI 리서치

목표는 **후보 30개를 AI Berkshire식으로 분석하고 점수화하는 것**입니다.

## Phase 2-1. AI Berkshire 설치 및 리서치 환경 구성

AI Berkshire README는 Claude Code 사용자의 경우 레포를 클론하고 Claude Code global commands 디렉터리에 설치하는 스크립트를 안내합니다. 또한 Codex 사용자를 위한 skill 설치 스크립트도 제공한다고 설명합니다.

리더님 프로젝트에서는 이렇게 나눕니다.

```
Claude Code- AI Berkshire 설치- /investment-team- /investment-research- /investment-checklist- /earnings-reviewCodex CLI- 리서치 결과 파싱- 점수표 생성- reports/ 통합
```

## Phase 2-3. 후보 30개 리서치 실행

실행 방식은 이렇게 잡으면 됩니다.

```
1. Codex가 screening_result.csv 생성2. Claude Code에서 후보 종목별 AI Berkshire 리서치 수행3. 각 종목 리포트를 reports/research/{ticker}_{name}.md로 저장4. Codex가 리포트 파일을 읽고 점수화 테이블 생성
```

처음에는 30개 전체를 한 번에 돌리지 말고, **3개 종목으로 파일 형식과 점수화 기준을 검증**해야 합니다.

# Stage 3 — 최종 선별

목표는 **후보 30개 중 실제 관심 universe 5~10개만 남기는 것**입니다.

## Phase 3-1. 정량 점수 + AI 점수 통합

점수는 다음처럼 분리하는 게 좋습니다.

```
Total Score = 정량 점수 50% + AI 리서치 점수 50%
```

또는 초기에는 더 보수적으로:

```
Total Score = 정량 점수 60% + AI 리서치 점수 40%
```

처음부터 AI 점수 비중을 너무 높이면 리포트 품질에 따라 결과가 흔들릴 수 있습니다.

이 단계는 “좋은 종목 고르기”보다 **나쁜 실수를 피하는 단계**입니다.

# Stage 4 — 매매 시스템

목표는 **관심 universe 안에서만 규칙 기반으로 진입/청산하는 것**입니다.

## Phase 4-1. 기술적 진입 조건 v1

처음부터 복잡한 전략을 쓰지 말고, 단순한 조건부터 시작합니다.

예시:

|조건|의미|
|---|---|
|가격이 20일선 위|단기 추세 확인|
|20일선이 60일선 위|중기 추세 확인|
|거래대금 증가|수급 확인|
|최근 고점 대비 과열 아님|추격매수 방지|
|시장 국면 위험 아님|하락장 회피|

---

## Phase 4-2. ATR 손절 구현

ATR 손절은 변동성 기반 손절입니다.

예시:

```
초기 손절가 = 진입가 - 2.5 × ATR(14)
```

단, 국내 주식은 상하한가, 호가단위, 장중 갭이 있으므로 백테스트에 이를 반영해야 합니다.

---

## Phase 4-3. 트레일링 스탑 구현

예시:

```
최고가 갱신 후청산가 = 최고가 - 3.0 × ATR(14)
```

트레일링 스탑은 수익을 길게 가져가는 데 유리하지만, 횡보장에서는 잦은 손절이 날 수 있습니다. 그래서 시장 국면 필터와 함께 봐야 합니다.

---

## Phase 4-4. 시장 국면별 비중 조절

초기에는 단순하게 갑니다.

|시장 국면|주식 비중|
|---|---|
|상승장|70~100%|
|중립장|30~70%|
|하락장|0~30%|

시장 국면 판단도 처음에는 단순한 기준이 낫습니다.

```
KOSPI가 200일선 위 = 위험 선호KOSPI가 200일선 아래 = 위험 축소KOSPI 20일선 < 60일선 = 신규 진입 제한
```

---

## Phase 4-5. 백테스트

백테스트는 반드시 매매 시스템보다 먼저 검증되어야 합니다.

반드시 반영할 것:

|항목|이유|
|---|---|
|수수료|실제 수익률 왜곡 방지|
|세금|국내 주식 비용 반영|
|슬리피지|체결가 오차 반영|
|거래대금 제한|비현실적 체결 방지|
|룩어헤드 방지|미래 데이터 사용 방지|
|생존자 편향 방지|현재 살아남은 종목만 쓰는 오류 방지|
|리밸런싱 주기|실전 운용과 일치|

---

## Phase 4-6. 실전 전 모의운용

실전 투입 전 최소 1~3개월은 모의운용을 해야 합니다.

모의운용 체크리스트:

```
- 오늘 매수 후보가 왜 나왔는가?- 손절가는 정상 계산되었는가?- 트레일링 스탑은 정상 갱신되는가?- 시장 국면 판단은 맞게 반영되는가?- 주문 가능 수량이 현실적인가?- 거래대금 대비 진입 규모가 과하지 않은가?- 리포트와 실제 신호가 충돌하지 않는가?
```
